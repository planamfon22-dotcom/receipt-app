# ระบบสะสมใบเสร็จสำหรับ Meet & Greet

โปรแกรมนี้ทำไว้สำหรับงานสะสมยอด 1 เดือน โดยลูกค้าส่งใบเสร็จผ่านลิงก์ `/submit` แล้วแอดมินเข้ามาตรวจสอบก่อนนับยอดสะสมจริง

## ฟีเจอร์หลัก
- ลิงก์สำหรับลูกค้ากรอกข้อมูลและอัปโหลดรูปใบเสร็จ
- แอดมินตรวจสอบใบเสร็จทีละรายการ พร้อมแก้ยอดอนุมัติได้
- สรุปยอดสะสมรายเดือนแยกตามลูกค้า
- จัดกลุ่มอัตโนมัติ
  - กลุ่ม 10,000 บาท = ยอดเกิน 5,000 แต่ไม่เกิน 10,000 บาท
  - กลุ่ม 5,000 บาท = ยอดเกิน 1,000 แต่ไม่เกิน 5,000 บาท
  - กลุ่มไม่เกิน 1,000 บาท
- สุ่มผู้โชคดีตามกลุ่มยอดสะสม
- Export CSV สำหรับเก็บหลักฐานหรือส่งทีมงานต่อ

## จุดที่ออกแบบไว้ให้ใช้งานง่ายและปลอดภัยขึ้น
- มีฝั่งลูกค้าและฝั่งแอดมินแยกกันชัดเจน
- ใช้ session login สำหรับแอดมิน
- รองรับการตั้งรหัสผ่านผ่าน environment variable
- ตรวจสอบชนิดไฟล์ก่อนอัปโหลด
- จำกัดขนาดไฟล์สูงสุด 8 MB
- ใช้ชื่อไฟล์ใหม่แบบสุ่ม ลดความเสี่ยงชื่อไฟล์ชนกัน
- มี CSRF token สำหรับฟอร์มหลัก

## วิธีรันในเครื่อง
1. เข้าโฟลเดอร์โปรเจกต์
2. สร้าง virtual environment และติดตั้งแพ็กเกจ

```bash
python -m venv .venv
source .venv/bin/activate   # Windows ใช้ .venv\Scripts\activate
pip install -r requirements.txt
```

3. ตั้งค่า environment variable

```bash
export SECRET_KEY='your-random-secret'
export ADMIN_USERNAME='admin'
export ADMIN_PASSWORD='your-strong-password'
```

4. รันระบบ

```bash
python app.py
```

จากนั้นเปิด:
- หน้าแรก: `http://127.0.0.1:5000/`
- ลิงก์ลูกค้า: `http://127.0.0.1:5000/submit`
- แอดมิน: `http://127.0.0.1:5000/admin/login`

## การนำขึ้นออนไลน์เพื่อให้มีลิงก์จริง
ถ้าต้องการลิงก์ให้ลูกค้าใช้งานจริง แนะนำ deploy ขึ้นบริการเช่น Render, Railway, Fly.io หรือ VPS ภายในองค์กร

เมื่อนำขึ้นออนไลน์แล้ว จะได้ลิงก์ประมาณนี้
- ลูกค้า: `https://your-domain.com/submit`
- แอดมิน: `https://your-domain.com/admin/login`

## โครงสร้างการใช้งานแนะนำ
1. ลูกค้าซื้อสินค้าแล้วเข้าลิงก์ `/submit`
2. อัปโหลดใบเสร็จ + กรอกยอดซื้อ + กดส่ง
3. แอดมินเปิด `/admin/reviews`
4. ตรวจสอบความถูกต้อง แล้วกดอนุมัติหรือไม่อนุมัติ
5. ปลายเดือนเปิด `/admin/monthly-summary`
6. ไปที่ `/admin/draw` เพื่อสุ่มผู้โชคดีแยกแต่ละกลุ่ม

## หมายเหตุเรื่องกติกา
ตอนนี้ระบบตีความตามเงื่อนไขที่ให้มาแบบตรงตัว:
- มากกว่า 5,000 ถึง 10,000 = กลุ่ม 10,000 บาท
- มากกว่า 1,000 ถึง 5,000 = กลุ่ม 5,000 บาท
- ไม่เกิน 1,000 = กลุ่มสุดท้าย
- ถ้ามากกว่า 10,000 ระบบจะขึ้นว่า “นอกเงื่อนไขปัจจุบัน” เพื่อให้คุณตัดสินใจว่าจะเพิ่มกลุ่มใหม่หรือรวมกับกลุ่มเดิมภายหลัง

หากต้องการ ผมสามารถต่อยอดเวอร์ชันถัดไปให้มี:
- OTP ทาง SMS
- ส่งอีเมลแจ้งผลรับใบเสร็จ
- Dashboard แบบกราฟ
- หลายแอดมิน
- Audit log
- Export รายชื่อผู้โชคดีแบบสวยงาม
- เชื่อม Google Drive / Cloud Storage สำหรับเก็บไฟล์ใบเสร็จ
